Hdfs 海量数据存储的意义

随着数据量越来越大，一个操作系统存不下所有数据，只能分开多台机器存储， 但是不方便管理和维护，迫切需要一种系统来管理多台电脑上的文件，这就是分布式文件管理系统。hdfs只是分布式文件管理系统的一种。

HDFS的定义

HDFS (Hadoop Distributed File System), 它是一个文件系统，用于存储文件，通过目录树来定位文件；其次，它是分布式的，由荤多服务器联合其他实现功能，集群中的服务器有各自的角色

HDFS使用场景： 一次写入，多次读出的场景。 一个文件经过创建、写入和关闭之后就不需要改变。

HDFS的优点和缺点

不适合低延时数据访问， 比如毫秒级的存储数据，是做不到的。

  • 无法高效的对大量小文姐进行存储。

  • 存储大量小文件的话， 它会占用NameNode 大量的内存来存储文件目录和块信息。 这样是不可取的，因为NameNode 的内存总是有限的

  • 小文件存储的寻址时间会超过读取时间，他违反了HDFS的设计目标

不支持并发写入、文件随机修改。

  • 一个文件只能有一个写，不允许多个线程同时写
  • 仅支持数据 append(追加)， 不支持文件的随机修改。

HDFS文件块大小

HDFS中的文件在物理上是分块存储（Block），块的大小可以通过配置参数 (dfs.blocksize) 来规定，默认大小在 Hadoop2.x/3.x 版本是 128M, 1.x 版本是 64M

1） 如果寻址时间约为 10ms， 即查找到目标 block 的时间为10ms

2） 寻址时间为传输时间的 1% 时，则为最佳（专家）。

3） 传输时间为 10ms/0.01 = 1000ms = 1s

4） 目前磁盘传输速率普遍为 100MB/s

思考： 为什么块不能太大，也不能太小

1） HDFS块设置太小，会增加寻址时间，程序一直找块的开始位置；

2） 如果块设置太大了，磁盘传输数据时间会明显大于定位这个块开始位置所需的时间。导致程序在处理这块数据时，会非常慢

总结： HDFS大小依据硬盘传输速率设置

认识HDFS

  • HDFS是 GFS的克隆版

  • HDFS特点： 扩展性&容错性（冗余存储）&海量数量存储

  • 将文件切分为指定大小的数据块并以多副本的方式存储在多个机器上

  • 数据切分、多副本、容错等操作对用户是透明的
